## 結論と推奨構成

### 推奨運用構成（メモリ別）

| メモリ | 推奨エンジン | 対応言語 | 用途 |
|--------|-------------|---------|------|
| 8GB | NLLB-200 600M | 22言語 | 簡易接見会話のみ |
| 16GB | NLLB-200 1.3B + OPUS-MT | 22言語 | 接見・公判前整理向き |
| 24-32GB | NLLB-200 3.3B + 中型LLM | 22言語 | 公判廷・裁判員裁判 |
| **64GB** | **Qwen2.5-72B Q4_K_M** | **6言語高精度 + 22言語実用** | **裁判員裁判・複雑事件** |
| 128GB | Qwen2.5-72B Q8_0 / Qwen3-235B | 全言語最高精度 | 通訳人を超える精度 |

### 既知の限界

- タガログ語は Qwen2.5-72B 単独では誤訳・解説混入が発生
- 長文では一部意訳のブレあり（中間英語＋逆翻訳でカバー可能）
- パンジャーブ語・シンハラ語等の少数言語は未検証
- 専門スラング（薬物隠語等）はDEA出典辞書で補強済み（[DICTIONARY.md](./DICTIONARY.md)）

### 重要：誤訳予防の運用

どのモデルでも100%正確な翻訳は不可能です。PLIは以下の仕組みで誤訳を検知できる設計になっています：

1. **中間英語の併記表示** — 弁護人が `robbery` / `theft` を即座に判別
2. **逆翻訳の併記表示** — 依頼者に何が伝わるかを日本語で確認
3. **法律用語辞書の優先適用** — 重要用語は辞書で固定
4. **複数エンジン併用** — OPUS-MT / NLLB / LLM を切り替え
5. **全履歴の自動保存** — 後日の検証・控訴審での立証に使用可能
