## 1. 概要

PLI（Private Link Interpreter）は、留置施設の接見室における弁護人と被疑者・被告人間のコミュニケーションを支援する **完全オフラインAI通訳システム** である。

### 1.1 設計理念

- **完全オフライン動作**: 通信ログ・翻訳履歴が外部に漏洩する経路を物理的に遮断
- **秘匿性**: F1（ハイド）/ F2（パニック）による即時隠蔽・即時データ消去
- **弁護人主導**: 翻訳結果は弁護人が確認・承認してから被疑者に提示
- **信頼性**: 構文反転チェックによる翻訳精度の検証機構

### 1.2 対象ハードウェア

| 項目 | 仕様 |
|------|------|
| メインPC | MacBook Pro M3 Max 64GB |
| 被疑者ディスプレイ | 13インチポータブルモニター（USB-C） |
| マイク | USB外部マイク（推奨） |
| GPU推論 | Apple Metal（llama.cpp） |

## 3. コアコンポーネント

### 3.1 Interpreter（core/interpreter.py）

翻訳パイプラインの中核。STTとLLMを統合し、弁護人・被疑者双方の発言を翻訳する。

#### エンジン構成

| エンジン | 実装 | モック |
|---------|------|--------|
| **STT** | faster-whisper (large-v3-turbo) | MockSTT |
| **LLM** | llama.cpp (Qwen2.5-32B-Q6_K) | MockEngine |

#### 翻訳フロー

**弁護人 → 被疑者**（ストリーミング）:
1. 弁護人が日本語テキストを入力
2. LLMが対象言語にストリーミング翻訳
3. 被疑者ディスプレイにタイプライター表示

**被疑者 → 弁護人**（同期＋承認フロー）:
1. STTが被疑者の発言を認識
2. LLMが日本語に翻訳
3. **弁護人が OK / やり直し / 手動修正 を選択**
4. 承認後に会話ログに記録

#### 構文反転チェック

外国語文を句単位に分解し、日本語語順（SOV）で対訳テーブルを生成。弁護人が対訳を確認・修正し、修正後の文を再構成する。

### 3.2 Recorder（core/recorder.py）

| モード | 説明 | データ保持 |
|--------|------|-----------|
| **OFF** | 録音なし | — |
| **VOLATILE** | RAM上のみ | アプリ終了で消失 |
| **SAVE** | WAVファイル保存 | ~/pli-recordings/ |

- パニック時（F2）: バッファをゼロ上書き後に解放
- ハイド時（F1）: 設定に応じて消去

### 3.3 HideMode（core/hide_mode.py）

| キー | 動作 | データ処理 |
|------|------|-----------|
| **F1** | 画面隠蔽（トグル） | 設定に応じて消去 |
| **F2** | パニック（不可逆） | 全データ即時消去 |

F1/F2発動時:
- PLIウィンドウを非表示
- ダミーアプリ（PDF/Finder）を前面表示
- macOS Dock/Cmd+Tab から除外

## 4. UI仕様

### 4.1 弁護人コンソール（AttorneyWindow）

**カラースキーム**:
- 背景: #1a1a2e（紺色）
- パネル: #16213e
- 弁護人発言: #4fc3f7（シアン）
- 被疑者発言: #81c784（緑）
- 警告: #ff8a65（オレンジ）
- 入力フィールド文字色: #4fc3f7（シアン）

**メニュー構成**:
| メニュー | 内容 |
|---------|------|
| **言語** | 24言語選択（ティア別セパレータ付き） |
| **セッション** | セッション終了・F1・F2 |
| **録音** | OFF / 揮発 / 保存 |
| **テスト** | 被疑者発言シミュレーション（Ctrl+D） |
| **設定** | ハイド時消去設定・ダミーPDF選択 |

**承認フロー**:
被疑者の発言に対して3つの選択肢:
- ✓ OK → 翻訳確定、ログに記録
- ✗ やり直し → 被疑者に再発話を求める
- ✎ 手動修正 → 構文反転チェックダイアログを開く

### 4.2 被疑者ディスプレイ（DefendantWindow）

**表示専用** — 被疑者側に操作UIは一切ない。

**メッセージ表示**:
- 弁護人発言: 青背景（#1c2333）タイプライター表示
- 被疑者発言: 緑背景（#1a2a1a）即時表示
- テキストサイズ: 22px（13インチモニターで視認性確保）

**ステータスバナー** （20px太字、承認フロー状態表示）:
| 状態 | バナー表示 | 背景色 |
|------|-----------|--------|
| 確認待ち | ⏳ Verifying with your attorney... | #1a2a3a |
| 確定 | ✓ Confirmed（3秒後自動消去） | #1a2a1a |
| やり直し | ✗ Could not understand. Please say that again. | #2a2000 |
| 修正中 | ⏸ Your attorney is correcting the translation. | #2a2000 |
| 修正確定 | ✓ Corrected and confirmed（3秒後自動消去） | #1a2a1a |

| モデル | Qwen2.5-32B-Instruct |

## 7. 起動方法

## 9. セキュリティ設計

### 9.1 データライフサイクル

| データ | 保持場所 | 消去タイミング |
|--------|---------|---------------|
| 会話ログ | メモリ（Python list） | セッション終了 / F1 / F2 |
| 翻訳結果 | メモリ | 同上 |
| 録音（揮発） | RAM (BytesIO) | アプリ終了 / F1 / F2 |
| 録音（保存） | ~/pli-recordings/ | 手動削除 / F1設定時 / F2（当該セッション分） |
| LLMモデル | ~/pli-models/ | 恒久 |

### 9.2 パニック時の消去手順

1. 全ウィンドウ非表示
2. 録音バッファをゼロ上書き → 解放
3. 当該セッションで保存した録音ファイルを削除
4. 会話ログ消去
5. ダミーアプリを前面表示
6. Dock/Cmd+Tab から除外

### 9.3 ネットワーク隔離

- 推論: ローカルGPU（Metal）
- STT: ローカルCPU
- **外部通信は一切発生しない**
