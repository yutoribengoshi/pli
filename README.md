# PLI — Private Link Interpreter

> **ローカルLLMで動く、完全オフラインの法律通訳AI**
> 刑事弁護人のための、依頼者の発言を一切外部に送信しない通訳ツール

開発者: [中野通り法律事務所 弁護士 関 智之](https://github.com/yutoribengoshi)

---

## はじめに：「ローカルLLM」とは何か

弁護士の先生方には聞き慣れない言葉かと思いますので、まず簡単にご説明します。

### LLMとは

**LLM** は Large Language Model（大規模言語モデル）の略で、ChatGPT・Claude・Gemini等の名前で知られているAI技術のことです。文章を理解したり、翻訳したり、要約したりする能力を持ちます。

### 「ローカル」とは

通常、ChatGPT等のLLMはクラウド（外部のデータセンター）で動いています。利用者がスマホやパソコンに入力した文章は、必ずインターネット経由で外部のサーバー（OpenAI社、Google社等）に送信され、そこで処理されてから結果が返ってきます。

これに対し**ローカルLLM**は、**手元のパソコン本体の中**でLLMを動かす方式です。文章は外部に送信されず、すべての処理がMacの内部で完結します。

```
[ChatGPT等の通常LLM]              [PLIのローカルLLM]
                                  
   依頼者の発言                    依頼者の発言
       ↓                              ↓
   インターネット                  Mac本体内のAIが処理
       ↓                              ↓
   米国OpenAI社サーバー            翻訳結果を画面表示
       ↓                              
   翻訳結果が返ってくる            （外部送信ゼロ）
```

### なぜ弁護士業務でローカルLLMが重要か

クラウドLLMに依頼者の発言を入力した瞬間、そのデータは：
- 外部企業のサーバーに保存される
- 海外（米国・EU等）に送信される
- AI学習データに使われる可能性がある
- 該当国の捜査機関が令状で取得できる

**「裁判で否認している依頼者の発言が、米国捜査機関の令状でアクセス可能なサーバーにある」状態は、弁護士法23条（守秘義務）に照らし許容できません。**

ローカルLLMであれば、これらの問題が発生しません。

### PLIで使用しているローカルLLM

PLIは以下のオープンソースのローカルLLMを組み合わせて動作します。すべて手元のMac内で動きます。

| モデル名 | 開発元 | 役割 |
|---------|--------|------|
| **Qwen3.5-9B**（標準） | Alibaba（中国） | 翻訳LLM。201言語対応・16GB機で0.5秒/文 |
| **Qwen2.5-72B**（高精度オプション） | Alibaba（中国） | 翻訳LLM。7言語で法律用語検証済み・64GB機向け |
| **NLLB-200** | Meta（米国） | 200言語対応の機械翻訳特化モデル |
| **OPUS-MT** | ヘルシンキ大学 | 言語ペア専用の軽量翻訳モデル |
| **Whisper-turbo** | OpenAI（米国） | 音声認識モデル（法律語彙バイアス付き） |

これらは全て「学術研究目的で公開されたAIモデル」で、PLIはダウンロードしたモデルファイルをMac内で実行しているだけです。実行中も外部通信は発生しません。

## なぜ作ったか

外国人刑事事件を弁護していると、通訳を巡って次のような問題に直面します。

- 「非弁通訳」が依頼者を取り込もうとする
- 強引な通訳人が、依頼者に勝手なことを吹き込む
- 法テラスから通訳費用の立替を命じられ、結果として弁護費用が高くつく
- 通訳人が法廷の柵（バー）に入れず、弁護権が事実上制限される
- そもそも稀少言語の通訳人が見つからない

**「依頼者と弁護人の間に、信頼できる通訳が常に介在する状態」を、外部の通訳人に頼らずに作る。**

これがPLIの目的です。

## 誤訳予防の仕組み

通訳AIで弁護人が一番怖いのは「気付かないうちに依頼者に誤った内容が伝わっていること」。
PLIはこれを最重視して以下を実装しています。

### 1. 中間英語の併記表示
```
依頼者発話 → 母語認識 → 英語に翻訳 → 弁護人言語に翻訳
                            ↑ ここを弁護人画面に常時表示
```
弁護人が `robbery` と `theft` のように **英語レベルで誤訳を即座に検知** できます。

### 2. 逆翻訳の併記表示
弁護人の発言を依頼者の母語に翻訳した後、もう一度日本語に逆翻訳して併記します。
**依頼者に何が伝わるかを日本語で確認** できます。

### 3. 法律用語辞書の翻訳プロンプト動的注入（4,339語）
内蔵の法律用語辞書（[DICTIONARY.md](docs/DICTIONARY.md)）から、**発話に実際に登場した
法律用語だけを翻訳AIへ「この訳語を使え」と毎回注入**します（RAG方式）。
例:「覚醒剤取締法」→ AIが素で訳すと誤訳（向精神薬の法律名と混同）するところを、
辞書注入により正式訳 *Stimulant Drugs Control Act* に固定。
- 法務省 日本法令翻訳（JLT）3,769語
- 関 智之 手動追加 570語（罪名・量刑・薬物スラング・関税法・入管法等）

### 4. 音声認識にも法律語彙バイアス
Whisper音声認識に法律用語の語彙ヒントを常時与え、同音異義語の誤認識を抑制します。
実測で日本語法律フレーズの文字誤り率を**54%削減**
（黙秘権←目比券、保釈←補釈、示談←時短、前科←善化、勾留←交流 等が正しく変換）。

### 5. 同音異義語のワンタップ訂正
「接見↔石鹸」「勾留↔交流」のような紛らわしい同音異義語が発話に含まれると、
バブルに🔄マークを表示。タップで別候補に差し替え→自動で再翻訳します。

### 6. 未知語のハイライト
辞書にない固有名詞・俗語・方言を画面で警告し、弁護人が確認するまで翻訳を確定させない設定が可能。

### 7. 全履歴の保存
原文・中間英語・相手言語・逆翻訳の4つをタイムスタンプ付きで保存可能（ローカルのみ）。
**控訴審での通訳の正確性立証** にも使えます。

### 8. 複数翻訳エンジンの併用
- **ローカルLLM（標準）**：Qwen3.5-9B（高速）／ Qwen2.5-72B（最高品質）
- **NLLB-200**：200言語をカバーするMeta社の翻訳特化モデル
- **OPUS-MT**：言語ペア専用に訓練された軽量モデル

用途・言語ペアに応じて使い分け、または併用します。

## なぜ「完全オフライン」なのか

### 弁護士法23条（守秘義務）の問題

ChatGPT・Google翻訳・DeepL等のクラウドAIに依頼者の発言を送信した瞬間、
そのデータは外部サーバーに保存され、海外（米国・EU）の管轄下に出ます。

- AI学習データに使われる可能性
- 該当国の法執行機関が令状で取得可能
- 通信会社・サーバー事業者にメタデータが残る

**「裁判で否認している依頼者の発言が、米国捜査機関の令状でアクセス可能なサーバーにある」状態は、弁護士として許容できません。**

### 接見室・法廷ではそもそも繋がらない

| 場所 | Wi-Fi | 4G/5G |
|------|:---:|:---:|
| 拘置所 接見室 | ✗ | △（電波弱い・地下も多い） |
| 警察署 接見室 | ✗ | △ |
| 法廷 弁護人席 | ✗ | △ |
| 公判前整理手続室 | ✗ | △ |

**「繋がっても繋いではいけない」+ 「肝心の現場で繋がらない」のダブルで、オフライン設計が必須です。**

## 動作環境

PLIは「Mac本体内ですべてのAIを動かす」設計のため、**搭載メモリの大きさが翻訳精度・処理速度に直結** します。

Apple Silicon Macは、CPUとGPUがメモリを共有する「ユニファイドメモリ」を採用しているため、**メモリ容量＝AI処理性能** と考えてください。

| メモリ | 動作レベル（Mac / Apple Silicon） |
|--------|----------|
| 8GB | 軽量LLM（Qwen3.5-4B）+ 辞書注入。簡易な接見会話向き |
| **16GB ★** | **標準構成（Qwen3.5-9B + 辞書4,339語注入）。GPU 0.5秒/文・全往復2.6秒** |
| 32GB | 標準構成に余裕。長時間・長文でも安定 |
| **64GB** | **高精度オプション（Qwen2.5-72B Q4_K_M）も選択可。7言語検証済み** |
| 128GB | 最大構成（Qwen2.5-72B Q8_0等）。微妙なニュアンスまで対応 |

### 推奨機種

| 用途 | 機種 | 価格 |
|------|------|---:|
| 大半の刑事弁護用途 | MacBook Air M3 16GB | 約20万円 |
| 裁判員裁判・複雑事件 | MacBook Pro M4 Max 64GB | 約60万円〜 |

### Windows版について

Windows版はコード対応済み（実機検証はこれから）です。Qwen3.5世代の小型・MoEモデルの
登場で、**NVIDIA GPUなしの一般的な業務ノートでもCPU推論で実用圏**になりました:

| メモリ | 構成 | 速度目安（CPU推論・推定） |
|--------|------|------|
| 8GB | Qwen3.5-4B + 辞書注入 | 5〜10秒/文 |
| 16GB | Qwen3.5-9B + 辞書注入 | 4〜8秒/文 |
| 32GB | Qwen3.6-35B-A3B (MoE) | 検証予定 |

小型モデルでも辞書注入により法律用語の精度は維持されます（Mac CPUでの実測で確認済み）。
ただし接見室・法廷での静音性・バッテリー・携行性では、ファンレス・1.2kg・15時間駆動の
MacBook Airが依然として最適です。
